1. Overview
In this assignment, you will build your own Routing Daemon from scratch using Python. Your
software will implement a simplified version of a Distance-Vector protocol (similar to RIP) to
automatically discover network topologies and calculate the shortest paths between subnets.
You will deploy your code inside Docker containers, which will act as the physical routers in a
virtual lab environment.

3. The Lab Topology
Students will work in teams to build a triangle topology. Each "Router" (Container) is connected
to two distinct Docker networks.
‚óè‚Äã Router A: Connected to Net_AB (10.0.1.0/24) and Net_AC (10.0.3.0/24)
‚óè‚Äã Router B: Connected to Net_AB (10.0.1.0/24) and Net_BC (10.0.2.0/24)
‚óè‚Äã Router C: Connected to Net_BC (10.0.2.0/24) and Net_AC (10.0.3.0/24)‚Äã
The Goal: If Router A wants to send a packet to Router C, it should know to go through Net_AC
(1 hop). If you "disconnect" (stop) Router C, Router A must eventually learn that it can reach the
subnet behind C via Router B.
